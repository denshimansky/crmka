## Ключевые темы обсуждения

### 1. Глоссарий и единая терминология

Согласованы ключевые термины системы:

| Термин | Определение |
|--------|-------------|
| **Клиент** | Физлицо, которое платит за услуги (родитель в детском центре). Всегда = плательщик |
| **Подопечный** | Тот, кто получает услугу (ребёнок). Информационная сущность, не участвует в финансовой логике |
| **Лид** | Контакт на этапе воронки продаж до первой оплаты. До оплаты — «контакт/лид», после — «клиент» |
| **Абонемент** | Предоплата за определённое количество услуг. Привязан к клиенту, содержит инфо о подопечном, направлении, группе |
| **Направление** | Тип услуги (английский, скорочтение, стрейчинг). Определяет стоимость занятия |
| **Инструктор** | Сотрудник, проводящий занятия (вместо «педагог» — юридические риски из-за лицензирования) |
| **Группа** | Конкретный слот в расписании: направление + кабинет + время + инструктор + ученики |
| **Занятие** | Один конкретный урок/сеанс в расписании (дата, время, группа) |
| **Посещение** | Факт присутствия/отсутствия на занятии. Статусы: явка, прогул, перерасчёт |

### 2. Воронки продаж и работа с клиентами

- Нужны **две отдельные воронки**: первичные лиды (новые контакты) и существующие/бывшие клиенты (допродажи, возвраты)
- Бывшие клиенты не участвуют в расчётах маркетинговых отчётов
- Пробные занятия: поддержать и платные, и бесплатные варианты
- На одном лиде может быть несколько пробных на разные направления
- **Чёрный список**: при вводе номера — предупреждение; внесение только руководителем/управляющим с указанием причины
- Потенциальные клиенты остаются в базе бессрочно, не переводятся в архив автоматически

### 3. Абонементы и финансовая логика

- Период списания — **календарный месяц**. Каждый месяц = новый абонемент (не продление старого, важно для LTV)
- Абонементы генерируются из расписания, а не продляются автоматически
- Клиент может переплатить (остаток на балансе) или ходить в долг — система не блокирует
- Тревога — при приближении баланса к нулю (не хватает на следующее занятие)
- Возвраты — только полные, частичных не бывает. Оформляются через кассу
- **Скидки**: постоянные (инвалиды, многодетные, сотрудники), разовые (акции, «приведи друга»), связанные (второе направление, второй ребёнок — пересчёт при отчислении основания). Суммирование скидок — настраиваемо
- Справочник причин отчисления (стандартизированный + «другая причина») — для статистики

### 4. Расписание и посещения

- Расписание еженедельное, но дни и время могут различаться внутри одной группы (пн 15:00, ср 16:00)
- Генерация расписания: продление из шаблона на следующий месяц; август-сентябрь — формирование с нуля
- Праздники/каникулы: руководитель убирает дни из расписания, абонементы пересчитываются автоматически
- Статусы посещений: явка, прогул (100% списание), перерасчёт (0% списание), отработка (не списывается, но фиксируется)
- Отметка задним числом — нужна. Закрытие периода вручную раз в месяц, с возможностью для руководителя открыть закрытый период
- Замены инструкторов — частые (1-2 раза/месяц). Оплата по ставке заменяющего

### 5. Зарплаты и финансы

**Инструкторы:**
- Схема: фикс за выход + сумма за каждого ребёнка (вместо сложных вариаций)
- Разные ставки по направлениям и филиалам
- Изменение ставки — с указанием даты начала действия
- Оплата за пробное: только если пробное платное

**Другие сотрудники:**
- Админы: оклад/ставка за смену + опционально % от выручки + бонус за продажу (оплату нового клиента)
- Руководитель: оклад + % от выручки/прибыли
- Уборщицы: ставка за выход × количество дней
- Бухгалтерия: оклад
- Штрафы/премии: произвольные суммы с комментарием, датой, автором

**Финансовый результат:**
- Формат А — общий по предприятию, Формат Б — по направлениям/филиалам
- Маржа = выручка − переменные расходы (зарплата инструкторов привязана к оказанию услуги)
- Расходы: постоянные (аренда, коммуналка, связь) и переменные (по направлениям, кабинетам, инструкторам)
- Постоянные расходы копируются из месяца в месяц с возможностью корректировки
- Кассы: количество настраиваемое, перемещение между кассами, инкассация (нал ↔ безнал)

## Принятые решения

1. **Терминология**: «педагог» → «инструктор» (юридические риски лицензирования)
2. **Клиент = плательщик** всегда, контактные лица (бабушки/дедушки) — доп. номер в информационном поле
3. **Два номера телефона** у клиента достаточно, отдельные контактные лица не нужны на текущем этапе
4. **Две воронки** продаж: первичная (новые лиды) и вторичная (допродажи/возвраты существующим)
5. **Каждый месяц — новый абонемент**, не продление (для корректного LTV по количеству и сумме)
6. **Оплата инструктора**: фикс за выход + ставка за каждого ребёнка (единая схема вместо множества вариаций)
7. **Прогул = 100% списание** (50% не предусматривается для детских центров)
8. **Доступы по филиалам**: один, несколько или все (не «один или все»)
9. **История изменений** (audit log) — по всем объектам, в фоне, доступна при необходимости
10. **Чёрный список** — только руководитель/управляющий может внести, с обязательным указанием причины
11. **Замена инструктора** — оплата по ставке заменяющего, не основного
12. **Закрытие периода** — вручную раз в месяц, руководитель может переоткрыть
13. **Автозадачи** — отложены на следующий этап (сложный блок)
14. **Роли**: руководитель, управляющий (новая роль между руководителем и админом), администратор, инструктор, бухгалтер + роль «только чтение» (для инвесторов/партнёров)

## Открытые вопросы

1. **Название продукта** — «Умная CRM» не устраивает (CRM не отражает суть, ERP — непонятно клиентам). Нужно что-то простое и понятное. Домен «управленка» занят 1С-франчайзи
2. **Ценообразование и тарифы** — пока не сформулированы, Анна обещала подумать
3. **Шаблоны организации** (детский центр, школа, фитнес-студия) — нужны ли предзаполненные справочники из коробки
4. **Предзаполненные справочники из коробки** — статьи расходов, каналы привлечения, виды дней — нужно определить список
5. **KPI админов за продажи** — разделение входящие/исходящие, привязка к каналу — реализовывать ли в первой версии
6. **Бета-тестирование** — нужно определить клиентов-добровольцев для обкатки
