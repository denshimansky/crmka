## Основные темы

### 1. Проблемы с логикой воронки продаж
Дима в процессе разработки воронки начал задавать вопросы о переходах между этапами, которые обнажили пробелы в логике. Анна выявила конкретную проблему: непонятно, что происходит с лидом, у которого несколько детей, когда по одному ребёнку создана заявка, а по второму нет. Решение для «подвисших» заявок и лидов не было формализовано.

### 2. Структура воронки: лид → заявка → пробное
Анна описала три ключевых этапа воронки. **Лид** — родитель с телефоном, минимальные данные. **Заявка** — появляется подопечный, направление, филиал; у одного лида может быть несколько заявок на разных детей. **Пробное** — не все заявки доходят до этого этапа, часть отваливается. Логика переходов между этапами при нестандартных сценариях не проработана.

### 3. Разделение сущностей «клиенты» и «абонементы»
В системе количество абонементов превышает количество клиентов (например, 200 клиентов, 250 абонементов). Часть отчётов требует данных по клиентам, часть — по абонементам. Необходимо разделить эти сущности в системе и добавить соответствующую логику в воронку.

### 4. Работа по складу и сроки
Наташа приедет завтра и поможет Диме закрыть задачи по складу. По оценке Анны, это несложно и некритично. После завершения воронки планируется запустить Настю в тест.

---

## Ключевые решения

1. **Лид перестаёт быть лидом в момент первой покупки** — как только оформлен первый договор/абонемент, статус меняется на «клиент», вне зависимости от количества детей.
2. **Незакрытые заявки по другим детям при переходе в клиенты переходят в «допродажу»** — они не остаются в воронке лидов, а переезжают в отдельный блок работы с действующими клиентами (по аналогии: купил iPhone, заявка на iPad → допродажа).
3. **В системе необходимо разделить сущности «клиенты» и «абонементы»** — оба показателя нужны, так как используются в разных отчётах.
4. **Денис и Дима созваниваются отдельно** для разбора технических вопросов по воронке без участия Анны.

## Открытые вопросы

- **Лид с несколькими детьми**: если по одному ребёнку заявка создана и ушла в пробное, а по второму клиент сказал «перезвоните через неделю» — где должен «висеть» второй ребёнок и в каком статусе? (Решение про «допродажу» обозначено концептуально, но не проработано технически.)
- **Заявки, которые «никуда не пошли»**: что происходит с заявками, по которым клиент отказался — они закрываются, архивируются, остаются в воронке?
- **Итоговая логика переходов** между этапами лид → заявка → пробное при нестандартных сценариях (несколько детей, частичный отказ) требует финального согласования на встрече Денис + Дима.
