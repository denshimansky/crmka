# CRMka

## О проекте
CRM-система для детских центров и сферы услуг в целом. Миграция существующего продукта с платформы 1С на веб-технологии. Текущая база: 20 организаций, 3–5 пользователей в каждой, цель — 500 партнёров к концу 2026.

**Дедлайн MVP:** 1 июня 2026 (жёсткий — 1 августа 2026)
**Старт разработки:** 16 марта 2026
**Пилот:** апрель 2026

## Архитектурные решения
- **Мультитенантность:** одна БД, одна схема, `tenant_id` + Row Level Security
- **Лид/Клиент** — разные статусы одной сущности. Переход автоматический при первой оплате. Обратно в лида — НЕВОЗМОЖНО
- **Подопечный** — информационная сущность, не участвует в финансах
- **Абонемент** заменяет «Договор/Направление/Интерес лида». Каждый месяц — новый (для LTV)
- **Три типа абонементов:** календарный (MVP), фиксированный (v1.1), пакетный (v2)
- **Воронка лидов** (новые, маркетинговая атрибуция) + **работа с клиентской базой** (допродажи/возвраты, без маркетинга)
- **Инструктор** (техническая роль; отображаемое название настраиваемо: «педагог», «тренер» и т.д.)
- **Выручка** = отработанные занятия (не оплаты). **Оборот** = движение денег (ДДС)
- **Маржа** = Выручка − Переменные расходы. **Рентабельность** = Чистая прибыль / Выручка
- **Постоянные расходы** автораспределяются пропорционально выручке
- **Три схемы ЗП инструкторов:** за ученика, за занятие, фикс за выход + за ученика
- **Оплата инструктору за прогул** — настройка организации
- **Пять ролей:** владелец, управляющий (настраиваемые), администратор, инструктор, только чтение
- **Мягкое закрытие периода:** снимок + корректировки для владельца/управляющего с аудитом
- **Мягкое удаление (soft delete):** deleted_at во всех ключевых сущностях
- **Авторизация:** email для владельца, логин/пароль (латиница) для сотрудников — создаёт владелец
- **Внутренние операции:** выемки, инкассации, переводы между счетами — отдельная таблица AccountOperation (не Payment)
- **Ценообразование SaaS:** 5000₽/мес за филиал + допы (интеграции)
- **Биллинг:** автовыставление счетов, грейс-период 1 рабочий день, автоблокировка
- **Личные кабинеты:** партнёра (подписка/счета) + клиента (расписание/оплата) — в MVP
- **Уведомления:** in-app (колокольчик + виджеты дашборда), внешние каналы — после запуска
- **Домен:** umnayacrm.ru

### Справка «?» (PageHelp)
При любом изменении страницы CRM — обновить справку:
1. Файл текстов: `app/src/lib/page-help-content.ts`
2. Компонент: `app/src/components/page-help.tsx`
3. Ключ справки = путь страницы без `(dashboard)/` (например `crm/leads`, `finance/payments`)
4. **При добавлении новой страницы** — добавить ключ и тексты в `page-help-content.ts`, вставить `<PageHelp pageKey="..." />` рядом с h1
5. **При изменении логики страницы** — проверить и обновить текст справки
6. **При удалении страницы** — удалить ключ из `page-help-content.ts`

### CI/CD — проверка после push
После каждого `git push` в main:
1. Проверить статус CI: `gh run list --repo denshimansky/crmka --limit 1`
2. Если CI упал — **починить до следующей фичи** (приоритет выше новых задач)
3. Если на локальной машине нет RAM для `tsc`/Playwright — полагаться на CI, но всегда проверять результат
4. Не пушить больше 3 коммитов подряд без проверки CI
