## Шаги теста

### ЧАСТЬ 0: Создание организации через бэк-офис
1. Логин в бэк-офис как superadmin
2. Создать организацию «Звёздочка» с owner (логин/пароль)
3. Логин под owner Звёздочки
4. Создать 2 филиала через UI настроек
5. Создать 4 кабинета (по 2 в филиале, ёмкость 8)

### ЧАСТЬ 1: Инфраструктура
6. Проверить страницу настроек
7. Создать 3 направления (Танцы 800₽, Рисование 600₽, Английский 700₽)
8. Создать 4 педагога (Волкова, Медведев, Лисицына, Орлов)

### ЧАСТЬ 2: Клиенты

### ЧАСТЬ 4: Абонементы и оплаты
15. Создать абонемент для Ивановой
16. Создать оплату от Козлова (переплата 10000₽)

### ЧАСТЬ 5: Расходы, задачи, зарплата

### ЧАСТЬ 5.5: Посещения и скидки

## Что было найдено и починено

| # | Баг | Статус |
|---|-----|--------|
| 1 | Нет CRUD филиалов (кнопка-заглушка) | **Починено** — POST/PATCH/DELETE + UI диалог |
| 2 | Нет CRUD кабинетов (только seed) | **Починено** — POST/PATCH/DELETE + UI диалог |
| 3 | DialogTrigger не работает через Playwright | **Починено** — заменён на Button onClick в 5 компонентах |
| 4 | Нет создания owner при создании партнёра | **Починено** — расширен POST /api/admin/partners |
| 5 | Кнопка «+ Абонемент» — клик на disabled заглушку | **Починено** — селектор :not([disabled]) |

## Как запускать

```bash
cd app

# Локально (dev-сервер должен работать)
TEST_BASE_URL=http://localhost:3000 npx playwright test tests/mega-business-scenario.spec.ts --timeout=120000

# На dev-сервере
npx playwright test tests/mega-business-scenario.spec.ts --timeout=120000
```

Каждый прогон создаёт новую организацию с уникальным суффиксом (timestamp). Данные остаются в БД для ручной проверки.
