# Бэк-офис: внутренняя админка SaaS

> **Версия:** 0.1 (черновик)
> **Дата:** 10.02.2026
> **Статус:** Зафиксировано, детализация позже

---

## Что это

Внутренняя панель управления для команды CRMka. Не видна партнёрам. Управляет всем, что находится «над» уровнем отдельной организации.

**Партнёр** — организация (юрлицо/ИП), которая покупает подписку на нашу CRM. Сейчас 20 партнёров, каждый — изолированный тенант.

---

## Блоки бэк-офиса

### 1. Управление партнёрами

| Что | Детали |
|---|---|
| Карточка партнёра | Название, юрлицо, ИНН, контактное лицо, телефон, email |
| Статус | Активный / Приостановлен / Заблокирован / Архив |
| Дата подключения | Когда начал пользоваться |
| Тариф и биллинг | Текущий тариф, дата следующей оплаты, история платежей |
| Количество филиалов | Для тарификации (если будет от филиалов) |
| Количество пользователей | Для мониторинга |

### 2. Подписки и биллинг

- Тарифные планы (создание, изменение)
- Привязка партнёра к тарифу
- История оплат
- Автопродление / ручное продление
- Блокировка при неоплате (грейс-период?)
- Выставление счетов
- Акты, документооборот

### 3. Управление доступом нашей команды

| Роль | Доступ |
|---|---|
| **Superadmin** | Полный доступ ко всему, включая данные всех партнёров |
| **Поддержка** | Доступ к данным конкретного партнёра (по запросу/тикету), только чтение |
| **Разработка** | Доступ к логам, метрикам, дебагу. Без доступа к персональным данным |
| **Биллинг** | Управление тарифами, оплатами, счетами |

**Обязательно:** логирование каждого входа в данные партнёра (кто, когда, зачем).

### 4. Мониторинг

- Активность партнёров: последний логин, количество операций за период
- «Спящие» партнёры: не логинились > N дней
- Объём данных: клиенты, занятия, платежи по каждому партнёру
- Здоровье системы: uptime, время ответа, ошибки

### 5. Поддержка

- Тикеты от партнёров (или интеграция с внешней системой)
- История обращений по партнёру
- SLA (время ответа, время решения)
- База знаний (внутренняя)

### 6. Резервное копирование

- Автоматические бэкапы (расписание, хранение)
- Восстановление данных партнёра
- Экспорт данных партнёра (если уходит)
- Удаление данных партнёра (GDPR / закон о персональных данных)

### 7. Онбординг партнёра

- Статус прохождения wizard
- Прогресс настройки (справочники, расписание, импорт)
- Флаг «нужна помощь»
- Назначение ответственного из нашей команды

### 8. SaaS-аналитика

- **MRR** (Monthly Recurring Revenue)
- **Churn rate** партнёров
- **LTV** партнёра
- **Средний чек**
- **Conversion** (триал → оплата, если будет триал)
- **NPS** (опросы)

## Приоритизация

| Когда | Что |
|---|---|
| **MVP** | Superadmin-роль, базовая карточка партнёра, ручное управление доступом. Остальное — в таблице/ноушене |
| **v1.1** | Биллинг, мониторинг активности, логирование доступа |
| **v2** | Тикеты, SaaS-аналитика, автоматические бэкапы с UI |
