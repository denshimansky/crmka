# Спецификация экранов и интерфейсов

> **Источник:** PRD v2.1
> **Технологии:** Next.js App Router, shadcn/ui, Tailwind CSS
> **Принципы:** PWA, mobile-first responsive, настраиваемый дашборд по ролям

---

## Навигация

### Основное меню (sidebar)

Боковая панель, сворачивается в иконки на мобильных. Вверху — логотип и название организации. Внизу — аватар пользователя, переключатель филиала.

- **Главная** (дашборд)
- **CRM**
  - Воронка продаж
  - Клиентская база
  - Обзвон
- **Расписание**
- **Посещения**
- **Абонементы**
- **Финансы**
  - Оплаты
  - Расходы
  - Счета (кошельки)
  - Должники
  - ДДС
  - Финрез
- **Зарплата**
- **Склад**
- **Задачи**
- **Отчёты**
- **Настройки**

Пункты меню отображаются в зависимости от роли и настроенных прав. Инструктор видит только: Главная, Расписание, Посещения, Зарплата (если разрешено). Роль «Только чтение» видит Отчёты и Дашборд.

### Плавающая кнопка «+»

Быстрое создание: лид, оплата, расход, задача. Доступна на всех экранах.

### Хлебные крошки

На всех экранах кроме дашборда. Формат: `Раздел / Подраздел / Сущность`.

### Переключатель филиала

Dropdown в шапке. Фильтрует все данные на всех экранах. Опция «Все филиалы» для сводного просмотра.

## 1. Авторизация и онбординг

### 1.1. /login — Вход

**Доступ:** все (неавторизованные)

**Блоки:**
- Логотип и название системы
- Поле «Email»
- Поле «Пароль»
- Кнопка «Войти»
- Ссылка «Забыли пароль?»

**Взаимодействие:**
- При вводе неверных данных — сообщение об ошибке
- При успешном входе — редирект на `/dashboard`
- Если у партнёра заблокирован доступ (неоплата) — редирект на `/blocked`

---

### 1.2. /blocked — Блокировка за неоплату

**Доступ:** пользователи заблокированного партнёра

**Блоки:**
- Красная плашка «Оплата не поступила»
- Информация о задолженности (сумма, период)
- Кнопка «Оплатить онлайн»
- Контакт поддержки

**Взаимодействие:**
- При поступлении оплаты — автоматическая разблокировка и редирект на `/dashboard`

---

### 1.3. /setup — Мастер настройки (wizard)

**Доступ:** владелец (первый вход после создания организации)

**Шаги:**

1. **Организация** — название, юрлицо, ИНН, адрес, логотип
2. **Филиалы** — добавление филиалов (название, адрес)
3. **Кабинеты** — кабинеты по каждому филиалу
4. **Направления** — название, стоимость занятия, стоимость пробного, длительность
5. **Счета (кошельки)** — создание касс/счетов, привязка к филиалам
6. **Сотрудники** *(необязательный)* — ФИО, роль, контакты, привязка к филиалам. Можно внести часть, остальных позже
7. **Инструкторы** *(необязательный)* — направления, ставки ЗП. Можно настроить позже
8. **Расписание** *(необязательный)* — создание групп, шаблоны, генерация. Можно создать часть
9. **Импорт клиентов** — загрузка из файла (или скрипт миграции из 1С)
10. **Готово** — сводка настроек, кнопка «Начать работу»

**Взаимодействие:**
- Прогресс-бар сверху
- Кнопки «Назад» / «Далее» / «Пропустить»
- Каждый шаг можно пропустить и вернуться позже через Настройки
- По завершении — редирект на `/dashboard`

## 2. Главная (/dashboard)

**Доступ:** все роли (набор виджетов зависит от роли)

**Описание:** настраиваемый дашборд. Пользователь может добавлять, убирать и переставлять виджеты. Дефолтный набор зависит от роли.

### Виджеты для владельца/управляющего:

| Виджет | Содержание |
|---|---|
| Выручка за месяц | Сумма + график по дням, сравнение с прошлым месяцем |
| Оборот за месяц | Поступления денег, динамика |
| Маржа | Маржа и рентабельность текущего месяца |
| Должники | Количество должников + сумма общего долга, ссылка на список |
| Воронка | Количество лидов по статусам, мини-воронка |
| Заполняемость | % заполнения групп по направлениям |
| Задачи на сегодня | Список задач с приоритетом |
| Новые лиды | Количество за неделю + график |
| Отток | Количество выбывших за месяц (не купили следующий абонемент) |
| Неотмеченные занятия | Таблица по дням с листанием по датам, количество неотмеченных |
| Сверка кассы | Поступления за сегодня по счетам |

### Виджеты для администратора:

| Виджет | Содержание |
|---|---|
| Задачи на сегодня | Список: звонки, пробные, напоминания, ДР |
| Расписание на сегодня | Ближайшие занятия, кабинеты |
| Неотмеченные занятия | Список занятий без отметки посещений |
| Должники | Список с обещанной датой |
| Новые лиды | Количество + последние добавленные |

### Виджеты для инструктора:

| Виджет | Содержание |
|---|---|
| Расписание на сегодня | Мои занятия, время, кабинет, состав |
| Неотмеченные | Занятия, которые нужно отметить |
| ЗП за месяц | Начислено (если разрешено в настройках) |
| ДР учеников | Ближайшие дни рождения подопечных |

**Взаимодействие:**
- Клик по виджету — переход в соответствующий раздел
- Кнопка «Настроить дашборд» — режим редактирования: drag-and-drop виджетов, добавление/удаление
- Период виджетов — переключатель (день / неделя / месяц) где применимо

## 3. CRM

### 3.1. /crm/leads — Воронка продаж

**Доступ:** владелец, управляющий, администратор

**Режимы отображения:**

**Канбан-доска (по умолчанию):**
- Колонки: Новый → Назначено пробное → Посетил пробное → Ожидание оплаты
- Боковые колонки (сворачиваемые): Потенциал, Нецелевой, Чёрный список, Архив
- Каждая карточка: имя (или телефон), направление, дата контакта, дни в статусе
- Цветовая подсветка: просроченная дата контакта — красная, сегодня — жёлтая, будущая — зелёная
- Drag-and-drop между колонками

**Таблица:**
- Колонки: ФИО, телефон, статус, направление, филиал, канал, дата создания, дата контакта, менеджер, комментарий
- Сортировка по любому столбцу

**Фильтры (общие для обоих режимов):**
- Статус, филиал, направление, канал привлечения, дата создания (период), менеджер

**Кнопки:**
- «+ Новый лид» — открывает форму создания
- Переключатель вид: Канбан / Таблица (всегда доступен)
- При 500+ лидах в воронке → автопереключение на табличный вид (kanban становится нечитаемым)
- Экспорт (Excel / PDF)

**Взаимодействие:**
- Клик по карточке/строке — переход на `/crm/lead/:id`
- При перетаскивании в «Чёрный список» — модальное окно с обязательной причиной
- Сортировка по умолчанию: дата контакта (сегодняшние сверху)

---

### 3.2. /crm/clients — Клиентская база

**Доступ:** владелец, управляющий, администратор

**Блоки:**

**Вкладки:** Активные | Допродажа | Выбывшие | Все

**Таблица:**
- Колонки: ФИО, телефон, подопечные, направления, сегмент (Новый/Стандарт/Постоянный/VIP), баланс, дата последней оплаты, LTV, филиал
- Сортировка по любому столбцу
- Бейдж сегмента с цветом

**Фильтры:**
- Филиал, направление, сегмент, наличие долга, период последней оплаты

**Кнопки:**
- «+ Новый клиент» (создание без воронки — для существующих клиентов при миграции)
- Экспорт (Excel / PDF)
- «Объединить дубликаты» — поиск совпадений по телефону

**Взаимодействие:**
- Клик по строке — переход на `/crm/client/:id`
- Массовые действия: выбрать несколько → создать обзвон

---

### 3.3. /crm/lead/:id — Карточка лида

**Доступ:** владелец, управляющий, администратор

**Блоки:**

**Шапка:**
- Статус (бейдж с возможностью смены)
- ФИО (или «Без имени» + телефон)
- Кнопки действий: «Назначить пробное», «Создать абонемент», «Принять оплату»

**Левая колонка — информация:**
- Контактные данные: телефон, доп. телефон, email, соцсети
- Канал привлечения
- Дата создания, менеджер
- Подопечные (имя, возраст, ДР) — добавление/редактирование inline
- Желаемое направление, филиал

**Правая колонка — лента активности:**
- Хронологическая лента: комментарии, смены статуса, пробные, звонки
- Поле ввода нового комментария + дата следующего контакта
- Лента не удаляемая

**Нижний блок — пробные:**
- Таблица назначенных пробных: группа, дата, результат
- Кнопка «Назначить пробное» — выбор группы (с отображением свободных мест), даты

**Взаимодействие:**
- Смена статуса через dropdown в шапке
- «Назначить пробное» → модальное окно выбора группы и даты
- «Создать абонемент» → переход на форму создания абонемента
- «Принять оплату» → модальное окно оплаты
- При совпадении телефона с существующим — предупреждение
- При совпадении с чёрным списком — блокировка с причиной

---

### 3.4. /crm/client/:id — Карточка клиента

**Доступ:** владелец, управляющий, администратор (инструктор — ограниченная версия)

**Блоки:**

**Шапка:**
- ФИО, телефон, сегмент (бейдж)
- Общий баланс клиента
- Кнопки: «Принять оплату», «Создать абонемент», «Допродажа», «Возврат» (при наличии баланса → модальное окно возврата)

**Вкладки:**

**Вкладка «Обзор»:**
- Контактные данные (телефон, доп. телефон, email, соцсети)
- Подопечные (список с возможностью добавления)
- Активные абонементы (направление, группа, баланс, остаток занятий)
- LTV (по деньгам + по количеству абонементов)
- Канал привлечения

**Вкладка «Абонементы»:**
- Таблица: направление, группа, подопечный, тип, период, баланс, статус
- Фильтр: активные / закрытые / все
- Кнопка «+ Создать абонемент»

**Вкладка «Оплаты»:**
- Таблица: дата, сумма, способ, счёт, привязка к абонементу, кто принял
- Кнопка «+ Принять оплату»

**Вкладка «Посещения»:**
- Таблица: дата, группа, вид дня, сумма списания
- Фильтр по периоду и направлению

**Вкладка «Коммуникации»:**
- Лента (как в карточке лида): комментарии, звонки, смены статуса
- Ввод нового комментария + дата следующего контакта

**Версия для инструктора:**
- Только ФИ подопечных, возраст, ДР
- Без контактов, балансов, финансов

### 3.5. /crm/calls — Обзвон

**Доступ:** владелец, управляющий, администратор

**Блоки:**

**Список кампаний обзвона:**
- Таблица: название, дата создания, кол-во контактов, обзвонено, осталось, статус (активный / закрыт)
- Кнопка «+ Новый обзвон»

**Взаимодействие:**
- Клик по кампании — переход на `/crm/calls/:id`

### 3.6. /crm/calls/new — Создание обзвона

**Доступ:** владелец, управляющий, администратор

**Блоки:**
- Название кампании
- Фильтры для формирования списка:
  - Статус клиента (лид / активный / выбывший)
  - Филиал
  - Направление
  - Возраст подопечного (диапазон)
  - Сегмент
  - Период последнего контакта
- Предпросмотр списка (таблица с количеством)
- Кнопка «Создать обзвон»

---

### 3.7. /crm/calls/:id — Кампания обзвона

**Доступ:** владелец, управляющий, администратор

**Блоки:**
- Прогресс-бар: обзвонено / всего
- Таблица контактов: ФИО, телефон, база-источник, дата последней связи, комментарий, ДР подопечного, интерес, статус звонка
- Фильтр: не обзвонены / обзвонены / все

**Взаимодействие:**
- Кнопка «Позвонить» рядом с контактом — открывает мини-форму: результат звонка, комментарий, дата следующего контакта
- Результат сохраняется в историю коммуникации клиента
- Кнопка «Закрыть обзвон» → перевод в архив

## 4. Расписание

### 4.1. /schedule — Расписание (недельный вид)

**Доступ:** все роли (инструктор видит только свои группы)

**Блоки:**

**Панель управления:**
- Навигация по неделям: ← Пред. неделя | Текущая | След. неделя →
- Переключатель вида: по кабинетам / по инструкторам / по группам / по направлениям
- Фильтры: филиал, направление, инструктор, кабинет (комбинируемые)

**Сетка расписания:**
- Горизонталь: дни недели (Пн–Вс) с датами
- Вертикаль: строки (кабинеты / инструкторы / группы — зависит от вида)
- Ячейки-блоки занятий:
  - Название группы, время, инструктор
  - Заполняемость: число / лимит
  - Цвет по заполняемости: зелёный (100%), жёлтый (51–99%), голубой (<50%), серый (0)
  - Иконка если есть пробные

**Взаимодействие:**
- Клик по занятию → переход на `/schedule/lesson/:id`
- Клик по пустой ячейке → создание разового занятия (если есть права)
- На мобильном — свайп между днями

---

### 4.2. /schedule/groups — Управление группами

**Доступ:** владелец, управляющий

**Блоки:**

**Таблица групп:**
- Колонки: название, направление, филиал, кабинет, инструктор, дни/время, учеников / лимит, статус (активная / архив)
- Фильтры: филиал, направление, инструктор, статус

**Кнопки:**
- «+ Новая группа»
- «Генерировать расписание» (для выбранных групп)
- «Массовое копирование расписания»

---

### 4.3. /schedule/groups/new — Создание группы

**Доступ:** владелец, управляющий

**Поля формы:**
- Название группы
- Направление (выбор из справочника)
- Филиал → Кабинет
- Инструктор
- Лимит учеников
- Шаблон расписания: чекбоксы дней недели + время начала + длительность для каждого дня
- Дата начала

**Проверка конфликтов расписания:**
- При создании/редактировании группы → автоматическая проверка:
  - Кабинет занят в это время?
  - Инструктор занят в это время?
- Предупреждение (не блокировка) — пользователь может игнорировать и сохранить

**Кнопки:**
- «Создать» — создаёт группу
- «Создать и сгенерировать расписание» — создаёт + генерирует занятия на текущий месяц

---

### 4.4. /schedule/groups/:id — Карточка группы

**Доступ:** владелец, управляющий, администратор (чтение)

**Блоки:**

**Шапка:** название, направление, инструктор, кабинет, заполняемость

**Вкладки:**

**Вкладка «Расписание»:**
- Календарь занятий группы (месячный вид)
- Каждое занятие: дата, время, статус (запланировано / проведено / отменено), кол-во учеников
- Кнопки: «Генерировать на месяц», «Удалить занятие» (праздники)

**Вкладка «Состав»:**
- Таблица учеников: подопечный, клиент, абонемент, дни посещения (если индивидуальное расписание)
- Кнопки: «+ Записать ученика», «Перевести», «Отчислить»

**Вкладка «Настройки»:**
- Редактирование шаблона расписания
- Смена инструктора (разовая / постоянная)
- Архивирование группы

---

### 4.5. /schedule/lesson/:id — Занятие

**Доступ:** все роли

**Блоки:**

**Шапка:** группа, дата, время, кабинет, инструктор (с возможностью разовой замены)

**Тема и ДЗ** (заполняет инструктор, опционально):
- Поле «Тема занятия» — текстовое
- Поле «Домашнее задание» — текстовое
- Без отправки на email (в MVP)

**Таблица посещений:**
- Колонки: подопечный, вид дня (выпадающий список), сумма списания, ЗП инструктору, переключатель «Оплата инструктору»
- Кнопка «Отметить всех» → ставит «Явка» всем
- Ручная корректировка вида дня для каждого
- Лиды на пробном — отмечены бейджем «Пробное»

**Нижний блок:**
- Итого: явок, прогулов, перерасчётов, отработок
- Сумма списаний, сумма ЗП инструктору

**Взаимодействие:**
- Сохранение — автоматическое (при смене вида дня)
- Если период закрыт — поля заблокированы (кроме владельца/управляющего с пометкой «после закрытия»)
- Если превышен дедлайн отметки — предупреждение (для админа — блокировка)

---

### 4.6. Печать расписания

**Доступ:** все роли

**Блоки:**

- Кнопка «Печать» на экране расписания (`/schedule`)

### 7.7. /finance/pnl — Финансовый результат

**Доступ:** владелец, управляющий

**Вкладки:** Общий P&L | По направлениям | По филиалам | По группам

**Вкладка «Общий P&L» (Формат A):**
```
Выручка (отработанные абонементы)
─────────────────────────────────
Переменные расходы:
  Зарплата инструкторов
  Канцтовары и хозтовары (со склада)
  Прочие материалы
─────────────────────────────────
Маржа (сумма + %)
─────────────────────────────────
Постоянные расходы:
  [по статьям]
─────────────────────────────────
Чистая прибыль
Рентабельность (%)
```

**Вкладка «По направлениям / По филиалам» (Формат B):**
- Таблица: строки — направления/филиалы, колонки — выручка, переменные, маржа, постоянные (пропорционально), прибыль, рентабельность

**Вкладка «По группам» (Формат C):**
- Таблица: строки — группы (с указанием направления, инструктора, филиала)
- Колонки: выручка группы, ЗП инструктора, переменные расходы (канцтовары, пропорц. по занятиям), пропорциональные постоянные расходы, прибыль группы, рентабельность группы (%)
- Переменные расходы распределяются пропорционально количеству занятий группы
- Постоянные расходы распределяются пропорционально выручке каждой группы
- Позволяет выявить убыточные группы и оценить эффективность каждого слота

**Фильтры:** период (месяц/квартал/год), филиал

**Drill-down:** каждая сумма кликабельна → расшифровка

**Справка:** иконка «?» — описание расчёта, определения, примеры

- Подтверждение удаления/архивирования
- Внесение расхода

### Таблицы

- Сортировка по клику на заголовок столбца
- Пагинация (20 / 50 / 100 строк)
- Поиск по таблице (строка поиска)
- Выбор видимых колонок
- Массовые действия через чекбоксы

### Формы

- Валидация на клиенте (inline-ошибки под полями)
- Автосохранение черновиков
- Подтверждение ухода со страницы при несохранённых данных

### Уведомления

- Toast-уведомления: успех (зелёный), ошибка (красный), предупреждение (жёлтый)
- **Колокольчик в правом верхнем углу хедера:**
  - Красный badge с количеством непрочитанных уведомлений
  - Источники: просроченные задачи, неотмеченные занятия, должники
  - Клик → выпадающий список уведомлений с кратким описанием
  - Клик по уведомлению → переход к источнику (задача, занятие, карточка клиента)
  - Отметка «прочитано» по одному или все сразу

### Empty states (пустые состояния)

Каждый экран при отсутствии данных показывает информативную подсказку вместо пустой таблицы:

- Структура: иконка + заголовок + описание + кнопка действия
- Примеры:
  - Клиенты: «Добавьте первого клиента» → кнопка «+ Новый клиент»
  - Расписание: «Создайте расписание» → кнопка «+ Новая группа»
  - Расходы: «Внесите первый расход» → кнопка «+ Внести расход»
  - Отчёты: «Пока нет данных для отчёта» → описание, какие данные нужны
  - Воронка: «Добавьте первого лида» → кнопка «+ Новый лид»
- Подсказки контекстные — зависят от конкретного экрана

### Мобильная версия (PWA)

- Sidebar сворачивается в бургер-меню
- Таблицы → карточки на мобильных
- Канбан → вертикальный список с фильтром по колонке
- Расписание → один день + свайп
- Плавающая кнопка «+» всегда доступна
- **Мобильная адаптация отчётов:**
  - Сложные табличные отчёты (финрез, зарплата, ДДС) → упрощённый вид: карточки с ключевыми цифрами
  - Баннер: «Для детального просмотра откройте с компьютера»
  - Простые отчёты (воронка, должники) отображаются полноценно
