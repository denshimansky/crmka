## Реальные баги (требуют исправления)

### BUG-001: Подписка не пересчитывается при создании филиалов
- **Серьёзность:** Высокая (финансы)
- **Где:** Биллинг — подписка организации
- **Описание:** При создании партнёра подписка создаётся с `branchCount=1` и `monthlyAmount=5000₽`. При добавлении филиалов подписка НЕ пересчитывается. У Звёздочки 2 филиала, а в подписке стоит 5000₽ вместо 10000₽.
- **Ожидаемое поведение:** При создании/удалении филиала — автоматически обновлять `branchCount` и `monthlyAmount` в активной подписке (`branchCount × pricePerBranch`).
- **Затронутые файлы:** POST /api/branches (нужен хук после создания), BillingSubscription
- **Скриншот:** Карточка «Звёздочка-72838» в бэк-офисе: 2 филиала, тариф 5000₽

### BUG-002: Расчётный счёт — проверка после создания не видит новый счёт
- **Серьёзность:** Низкая (UI refresh)
- **Где:** /finance/cash — диалог создания счёта
- **Описание:** Счёт реально создаётся (API 201), но после `router.refresh()` DOM не обновляется мгновенно. Тест не видит новый счёт в списке. При ручной перезагрузке — виден.
- **Ожидаемое поведение:** После создания счёта список обновляется автоматически.
- **Затронутые файлы:** add-account-dialog.tsx → `router.refresh()` не триггерит ререндер server component

### BUG-003: Оплата — проверка после создания не видит сумму
- **Серьёзность:** Низкая (UI refresh)
- **Где:** /finance/payments — диалог создания оплаты
- **Описание:** Оплата создаётся (API 201), но после закрытия диалога страница не обновляется. Сумма не видна в таблице до перезагрузки.
- **Ожидаемое поведение:** Список оплат обновляется автоматически после создания.
- **Затронутые файлы:** add-payment-dialog.tsx → `router.refresh()`

### BUG-004: Расход — проверка после создания не видит в списке
- **Серьёзность:** Низкая (UI refresh)
- **Где:** /finance/expenses — диалог создания расхода
- **Описание:** Расход создаётся, но после закрытия диалога страница не обновляется.
- **Ожидаемое поведение:** Список расходов обновляется автоматически.
- **Затронутые файлы:** expenses page → `router.refresh()`

### BUG-005: Посещения — расписание недели не показывает занятия
- **Серьёзность:** Средняя (навигация)
- **Где:** /schedule — недельный вид
- **Описание:** После генерации расписания группы, на странице /schedule нет кликабельных ссылок на занятия (`a[href*='/schedule/lessons/']`). Генерация создаёт занятия на будущие даты, а недельный вид показывает текущую неделю.
- **Ожидаемое поведение:** Навигация по неделям + ссылки на занятия видны.
- **Возможная причина:** Шаблоны расписания создаются без конкретных дней (кнопка «Добавить день» кликается но не настраивается day/time), генерация создаёт на пустой шаблон.

## Исправленные баги (починены в этой сессии)

### FIXED-001: Нет CRUD филиалов
- **Было:** Кнопка «Филиал» в настройках — заглушка без диалога. Нет POST API.
- **Починено:** POST/PATCH/DELETE /api/branches + CreateBranchDialog в настройках.

### FIXED-002: Нет CRUD кабинетов
- **Было:** Нет API и UI. Кабинеты только через seed.
- **Починено:** POST/PATCH/DELETE /api/rooms + CreateRoomDialog в настройках.

### FIXED-003: DialogTrigger не работает через Playwright
- **Было:** base-ui DialogTrigger с render prop не срабатывает при Playwright .click().
- **Починено:** Заменён на `<Button onClick={() => setOpen(true)}>` в 5 компонентах: create-branch-dialog, create-room-dialog, client-tabs (абонемент), add-account-dialog (счёт).

### FIXED-004: Нет создания owner при создании партнёра
- **Было:** Бэк-офис создавал организацию без owner. Невозможно войти в CRM.
- **Починено:** POST /api/admin/partners принимает ownerLogin/ownerPassword, создаёт Employee с role=owner + автоматическую подписку.

### FIXED-005: Кнопка «+ Абонемент» кликает на disabled заглушку
- **Было:** На карточке клиента две кнопки «Абонемент»: disabled в шапке и рабочая во вкладке. Playwright кликал на disabled.
- **Починено:** Селектор `button:has-text('Абонемент'):not([role='tab']):not([disabled])`.
