# Биллинг — архитектура и решения

Решения приняты: 08.04.2026

## 1. Юридическая структура

**Юрлицо:** ИП Шиманский Д.А.
**Система налогообложения:** АУСН 8% (доходы)
**Регион:** Московская область (АУСН доступна)

### Лимиты АУСН
- Выручка: до 60 000 000 ₽/год
- Сотрудники: до 5 (трудовые договоры; ИП/самозанятые не считаются)
- НДС: не начисляется
- Отчётность: автоматическая (банк → ФНС)

### Когда менять
| Клиентов | Выручка/год | Действие |
|----------|-------------|----------|
| до 500 | до 30 млн | ИП АУСН 8% — текущая схема |
| 500-700 | 30-60 млн | Мониторим приближение к лимиту |
| 700+ | 60+ млн | Открыть ООО, выбрать УСН 6% или АУСН (если лимиты поднимут) |

## 2. Ценообразование

**Базовая цена:** 5 000 ₽/мес за филиал (без НДС)

### Периоды оплаты
| Период | Множитель | Скидка |
|--------|-----------|--------|
| 1 месяц | ×1 | — |
| 3 месяца | ×3 | Нет (пока) |
| 6 месяцев | ×6 | Нет (пока) |
| 12 месяцев | ×12 | Нет (пока) |

> Скидки за длительный период НЕ вводим на старте. Добавим позже, когда проверим unit-экономику и поймём LTV/churn.

### Формула расчёта
```
Сумма = Цена × Кол-во филиалов × Кол-во месяцев
```
Пример: 2 филиала × 5 000 ₽ × 6 мес = 60 000 ₽

### Изменение количества филиалов

**Добавление филиала в середине периода:**
- Доплата = Цена × Оставшиеся полные месяцы
- Выставляется доп. счёт автоматом
- Пример: оплатил 3 мес (апр-июн) за 1 филиал. В мае добавил 2-й → доплата 5 000 × 2 мес = 10 000 ₽

**Удаление филиала:**
- Пересчёт со следующего периода
- Возврат за текущий период НЕ производится (прописано в оферте)

## 3. Документооборот

### Оферта (заменяет индивидуальный договор)
- Публикуется на сайте: umnayacrm.ru/offer
- Акцепт = оплата счёта
- Содержит: предмет, стоимость, порядок оплаты, SLA, порядок приёмки, персональные данные

### Ключевой пункт оферты — приёмка без актов
> «Услуги считаются оказанными надлежащим образом и принятыми Заказчиком, если в течение 5 (пяти) рабочих дней после окончания оплаченного периода Заказчик не направил мотивированную претензию.»

Это стандартная практика для SaaS (Битрикс24, amoCRM, Мегаплан).

### Документы

**Обязательно (автоматом):**
- Счёт на оплату (PDF) — генерируется из CRM
- Оферта на сайте — 1 раз

**По запросу клиента (скачать в ЛК):**
- Акт оказанных услуг (PDF)
- Акт сверки (PDF)

**Не требуется (АУСН, нет НДС):**
- Счёт-фактура
- УПД

## 4. Способ оплаты — безналичный расчёт

### Почему НЕ ЮKassa для B2B
- Комиссия ~3.5% = 350 ₽ с каждых 10 000 ₽
- При 500 клиентах × 10к × 12 мес = **2.1 млн ₽/год** потерь
- Для B2B (ИП → ИП) стандарт — безнал (р/с → р/с), комиссия 0%
- 54-ФЗ чеки для B2B не обязательны

### Основной банк: Т-Банк
**Почему:**
- Счёт уже открыт
- API бесплатный для клиентов
- Лучшая документация среди банков РФ
- Поддержка выставления счетов, webhook при оплате, выписки

### Резервные банки (счета открыты)
- **Точка** — Public API + Pro API, хорошая документация
- **Альфа** — Alfa API, больше для крупного бизнеса
- **Озон Банк** — нет публичного API для счетов, не подходит

## 5. Интеграция с Т-Банк API

### Сценарий выставления и оплаты

```
CRM                          Т-Банк                     Клиент
 │                              │                          │
 ├─ Генерация счёта ──────────► │                          │
 │  (POST /invoice)             │                          │
 │                              ├─ Ссылка на оплату ─────► │
 │                              │  (email клиенту)         │
 │                              │                          │
 │                              │ ◄── Клиент оплачивает ── │
 │                              │     (онлайн-банк)        │
 │  ◄── Webhook ────────────── │                          │
 │  (статус: оплачено)          │                          │
 │                              │                          │
 ├─ Обновить подписку           │                          │
 ├─ Сгенерировать акт           │                          │
 └─ Отправить акт на email      │                          │
```

### Эндпоинты Т-Банк API
- `POST /invoice` — создать счёт
- `GET /invoice/{id}` — статус счёта
- `GET /bank-statement` — выписка (для сверки)
- Webhook — уведомление об оплате

### Fallback (если API недоступен)
- Суперадмин вручную отмечает «оплачено» в бэк-офисе по банковской выписке

## 6. Автоматизация биллинга

### Цепочка уведомлений
| Когда | Действие | Канал |
|-------|----------|-------|
| За 14 дней | «Подписка истекает через 2 недели» | in-app |
| За 5 дней | Счёт сформирован, отправлен | Email |
| В день оплаты | «Сегодня срок оплаты счёта №N» | Email |
| +1 рабочий день | Грейс-период, предупреждение | Email + in-app |
| +2 рабочих дня | Блокировка тенанта | Email + in-app |

### Блокировка
- billingStatus → "blocked"
- Доступен только /billing (ЛК партнёра)
- API возвращает 403 (реализовано в middleware)
- Данные НЕ удаляются, восстановление после оплаты мгновенное

### Разблокировка
- Оплата через Т-Банк → webhook → автоматическая разблокировка
- Или: суперадмин вручную разблокирует в бэк-офисе

## 7. Баланс партнёра в ЛК

### Что видит партнёр на /billing
- Текущий тариф: 5 000 ₽ × N филиалов = X ₽/мес
- Оплаченный период: выбранный (1/3/6/12 мес)
- Оплачено до: DD.MM.YYYY
- Баланс: X ₽ (переплата/долг)
- История операций (таблица):
  - Дата | Операция | Сумма | Статус
  - 01.04 | Счёт №12 | 10 000 ₽ | Оплачен
  - 15.05 | Доплата за филиал | 5 000 ₽ | Ожидает

### Когда возникает баланс
- Переплата (округлил сумму) → положительный баланс → зачтётся
- Доплата за филиал → отрицательный баланс → выставляется счёт
- Возврат → зачисление на баланс (не на карту, прописать в оферте)

## 8. Модель данных (изменения)

### Изменения в BillingSubscription
```
+ billingPeriodMonths  Int       @default(1)    // 1, 3, 6, 12
+ periodEndDate        DateTime                  // дата окончания оплаченного периода
```

### Новая таблица: BillingBalance
```
BillingBalance {
  id              UUID
  organizationId  UUID (FK → Organization)
  type            enum (payment, charge, adjustment, refund)
  amount          Decimal        // + пополнение, − списание
  description     String
  invoiceId       UUID? (FK → BillingInvoice)
  createdAt       DateTime
}
```

### Изменения в BillingInvoice
```
+ periodMonths     Int           // за сколько месяцев счёт
+ branchCount      Int           // филиалов на момент выставления
+ paymentUrl       String?       // ссылка на оплату (от Т-Банк API)
+ paidVia          enum? (bank_transfer, tbank_api, manual)
```
