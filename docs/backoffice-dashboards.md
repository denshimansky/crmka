## 1. Управленческий дашборд (ГОТОВ)

**Путь:** `/admin/dashboard`
**API:** `GET /api/admin/dashboard`

### Виджеты:
- **Партнёры** — всего / активных / грейс / заблокированных
- **Новые за месяц** — количество + рост в % vs прошлый месяц
- **MRR** — сумма активных подписок + количество подписок
- **Неоплаченные счета** — количество + сумма + просроченные
- **Топ-10 по клиентам** — таблица: партнёр, клиенты, сотрудники, филиалы, статус
- **Спящие партнёры** — нет активности 7+ дней (по AuditLog)
- **Не завершили онбординг** — список с датой регистрации

## 2. Финансовый дашборд (ПЛАНИРУЕТСЯ)

**Рекомендуемое время:** когда будет 50+ партнёров

### Виджеты:
- **MRR** — текущий + динамика по месяцам (график)
- **ARR** — MRR × 12
- **Churn rate** — % ушедших партнёров за месяц (подписка cancelled или blocked)
- **ARPU** — средний чек на партнёра (MRR / активных партнёров)
- **LTV** — ARPU / churn rate
- **Дебиторка** — сумма неоплаченных счетов, разбивка по срокам просрочки (1-7д, 8-30д, 30+д)
- **Динамика MRR** — график: новый MRR + expansion - churn - contraction
- **Прогноз выручки** — на следующий месяц (текущие подписки + тренд)
- **Retention по когортам** — % оставшихся партнёров по месяцу подключения

### Источники данных:
- BillingSubscription (status, monthlyAmount, createdAt, cancelledAt)
- BillingInvoice (status, amount, dueDate, paidAt)
- Organization (createdAt, billingStatus)

## 3. Технический дашборд (ПЛАНИРУЕТСЯ)

**Рекомендуемое время:** когда нагрузка станет вопросом (100+ партнёров)

### Виджеты:
- **API requests/min** — по тенантам (топ-10 самых нагруженных)
- **Ошибки 5xx** — за последние 24ч, группировка по endpoint
- **Время ответа API** — p50 / p95 / p99 (требует middleware логирования)
- **Размер БД** — по тенантам (pg_total_relation_size)
- **Количество сущностей** — по тенантам (клиенты, абонементы, занятия, оплаты)
- **Последний логин** — дата последнего входа по каждому тенанту
- **Rate limit hits** — кто упирается в лимит 100 req/min
- **Uptime** — время работы сервера, последний деплой

### Требуется для реализации:
- Middleware для логирования request/response time
- Таблица ApiRequestLog (tenantId, endpoint, statusCode, durationMs, createdAt) или внешний сервис (Grafana/Prometheus)
- Cron для очистки логов старше 30 дней

## Impersonation: «Войти как партнёр»

### Текущая реализация (вариант A)

**API:** `POST /api/admin/partners/[id]/impersonate`
**Доступ:** superadmin, support
**Механизм:** создаёт NextAuth JWT-токен owner-а, устанавливает cookie, открывает CRM в новой вкладке.
**Срок:** 1 час (не 24ч, для безопасности).
**Аудит:** запись в AuditLog (action: "impersonate", adminEmail, targetOrg, targetOwner).
**Плашка:** жёлтый баннер «Режим поддержки» в CRM с кнопкой «Выйти из режима».

### Будущая реализация (вариант C — с согласием)

Для повышения доверия партнёров:

1. Суперадмин нажимает «Запросить доступ» в бэкофисе
2. Партнёру (owner) приходит уведомление: «Поддержка запрашивает доступ к вашей CRM»
3. Партнёр нажимает «Разрешить» (или «Отклонить»)
4. При разрешении — создаётся временный доступ на 1 час
5. Все действия логируются с маркером impersonation
6. Партнёр может отозвать доступ в любой момент

**Требуется:**
- Модель ImpersonationRequest (adminId, organizationId, status: pending/approved/rejected/expired, expiresAt)
- Уведомление партнёру (in-app или email)
- UI «Запросы доступа» в ЛК партнёра
- Периодическая проверка expiration
