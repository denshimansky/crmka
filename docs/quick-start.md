# Быстрый старт: основные сценарии

Пять базовых операций для администратора. Названия кнопок и полей — как в интерфейсе.

---

## 1. Завести лида

**Откуда:** страница `/crm/contacts` → вкладка «Лиды».
Альтернатива — плавающая круглая кнопка «➕ Новый лид» (есть на дашборде и в воронке).

**Шаги:**
1. Нажать кнопку **«➕ Клиент»** в верхней панели вкладки.
2. В диалоге «Новый клиент» (или «Новый лид» — если через плавающую кнопку) заполнить:
   - **Фамилия**, **Имя** — обязательно;
   - **Телефон** или **Соцсеть** — обязательно хотя бы одно из двух;
   - **Канал** (откуда узнал), **Филиал**, **Ответственный** — желательно;
   - **Подопечный** (ФИО ребёнка, дата рождения) — можно сразу добавить здесь.
3. Если система найдёт похожий контакт по телефону — покажет жёлтый блок «Найден похожий контакт». Открыть его, проверить — это не дубль.
4. Нажать **«Создать»**.

> Лид заводится со статусом «new». В клиента он перейдёт автоматически после первой полной оплаты.

---

## 2. Создать пробное

**Предусловие:** у лида должна быть **активная заявка** (Application) по нужному направлению. Без заявки кнопка пробного будет недоступна.

**Откуда заводится заявка:** в карточке клиента, блок **«Заявки»** → кнопка **«Создать заявку»** → выбрать подопечного, направление, филиал.

**Шаги для пробного:**
1. Открыть карточку клиента → вкладка **«Подопечные»**, либо страницу `/crm/sales` → вкладка «Пробное».
2. Нажать **«📅 Записать на пробное»** (на подопечном или на строке в продажах).
3. В диалоге «Запись на пробное занятие»:
   - если у ребёнка несколько активных заявок — сначала выбрать, по какой записываем (направление подставится автоматически);
   - выбрать **группу** (для записи в существующее занятие) **или** дату/время для индивидуального пробного;
   - указать **дату пробного**.
4. Нажать **«Записать»**.

> После проведения пробное переходит в статус «проведено», лид попадает на вкладку «Пробное проведено» в `/crm/sales`.

---

## 3. Завести группу в расписании

**Откуда:** страница `/schedule/groups`.

**Шаги:**
1. Нажать **«➕ Группа»**.
2. В диалоге «Новая группа» заполнить:
   - **Название** (пример: «Развивайка 3–4 (Пн/Ср)»);
   - **Направление**;
   - **Филиал** → **Кабинет** (список кабинетов фильтруется филиалом);
   - **Педагог** (список фильтруется филиалом);
   - **Макс. учеников** (по умолчанию 15);
   - **Дата старта** (пусто = сегодня), **Дата окончания** (пусто = +1 год);
   - **Ставка группы** — кнопка «Изменить» открывает выбор схемы ЗП (если отличается от стандартных ставок педагогов).
3. Блок **«Расписание»** → нажать **«➕ Добавить день»** для каждого занятия в неделю:
   - день недели, время начала, длительность в минутах.
   - Без хотя бы одного дня группа не создастся.
4. Если кабинет занят в это время — система покажет предупреждение «Кабинет уже занят». Сменить кабинет/время.
5. Нажать **«Создать»**. Расписание сгенерируется на год вперёд автоматически.

---

## 4. Внести оплату от родителя

**Откуда:** страница `/finance/payments` или карточка клиента → вкладка **«Оплаты»**.

**Шаги:**
1. Нажать **«➕ Оплата»**.
2. В диалоге «Новая оплата»:
   - выбрать **клиента** (если открыт из общей кассы) — иначе подставлен текущий;
   - сумма, дата, **счёт** (касса / банк / ЮKassa и т.п.);
   - комментарий (необязательно).
3. Нажать **«Создать»**.

> Деньги попадают **на баланс родителя**, а не сразу в абонемент. Чтобы зачесть оплату в конкретный абонемент — см. сценарий 5.

---

## 5. Создать абонемент

**Откуда:** карточка клиента → вкладка **«Абонементы»**.

**Шаги:**
1. Нажать **«➕ Абонемент»**.
2. В диалоге «Новый абонемент»:
   - **Направление** → **Группа** (фильтруется направлением);
   - **Подопечный** (если у клиента несколько детей);
   - **Год** + **Месяц** (для календарного абонемента) или **Дата начала** + **Срок (дн.)** (для пакетного);
   - **Цена занятия** (подставляется из направления, можно изменить);
   - **Занятий** — для календарного считается автоматически по расписанию группы;
   - **Скидка** (если есть).
   - В блоке «Итого» сразу видна полная стоимость и стоимость со скидкой.
3. Если на старом абонементе остался положительный баланс — увидите синюю подсказку «Есть остаток…»; перенести можно после создания кнопкой **↔** в таблице абонементов.
4. Нажать **«Создать»**. Абонемент появится со статусом **«ожидает оплаты»**.

---

## 6. Списать оплату в абонемент (закрыть долг с баланса)

**Откуда:** карточка клиента → вкладка **«Абонементы»**, строка нужного абонемента с долгом.

**Шаги:**
1. В строке абонемента нажать кнопку **«Списать»** (видна, пока есть долг по абонементу).
2. В диалоге «Оплатить с баланса родителя» подтвердить сумму (по умолчанию — полный долг абонемента, не больше доступного баланса).
3. Нажать **«Списать»**.

> Сумма уходит с баланса родителя и закрывает долг по абонементу. Если баланса не хватает — сначала внесите оплату (сценарий 4). После полной оплаты первого абонемента лид автоматически становится клиентом.

---

## Стандартный путь нового ребёнка

Лид → Заявка → Пробное → Создание абонемента → Оплата от родителя → Списание оплаты в абонемент → Посещение занятий (автосписание стоимости занятия с абонемента при отметке «Был»).
